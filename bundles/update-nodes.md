# Bundle: Update Nodes

After writing or editing a chapter, identify new canon and propose node updates.

## How to invoke

Provide: the chapter or scene text.
Output: proposed diffs per node — what to add, change, or remove, with reason. Wait for confirmation before editing any file.

## Process

1. Identify all characters, locations, and systems referenced in the scene.
2. Load their nodes.
3. Identify new information established: appearance details, rank changes, relationship shifts, Domain use, timeline events.
4. Cross-reference against existing node content.
5. For each delta: output `node-id → field → proposed change` and why.

## Rule

Never auto-apply. Always present as proposals. User confirms before any node is edited.
